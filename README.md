# Baika

Desktop stock screener for MOEX shares using PySide6, pandas, and MOEX ISS API.

## Развертывание (Команды для отчета)
1. Установка зависимостей:
   ```bash
   poetry install
    ```

2. Форматирование и проверка кода:
    
    
    
    ```Bash
    poetry run ruff format .
    poetry run ruff check .
    ```
    
3. Запуск тестов:
    
    
    
    ```Bash
    poetry run pytest
    ```

## 📂 Структура проекта (Project Structure)

```text
moex-screener/
├── .github/                       # Конфигурации CI/CD (опционально)
├── data/                          # Слои работы с физическими данными
│   └── fixtures/                  # Изолированные JSON-пакеты для тестирования и офлайн-разработки
│       ├── securities_corrupted.json       # Граничный случай: поврежденный JSON-пакет
│       ├── securities_empty.json           # Граничный случай: пустой ответ API (выходной день)
│       ├── securities_invalid_types.json   # Граничный случай: аномальные типы данных
│       ├── securities_missing_columns.json # Граничный случай: отсутствие критических колонок
│       ├── securities_real.json            # Актуальный слепок живого ответа MOEX ISS
│       └── securities_valid.json           # Базовый позитивный сценарий (SBER, GAZP и null-значения)
├── logs/                          # Каталог автоматического ведения логов (создается динамически)
│   └── moex_screener.log          # Асинхронный ротируемый лог-файл приложения
├── src/                           # Исходный код приложения (Архитектура MVC/MVVM)
│   ├── __init__.py
│   ├── main.py                    # Главная точка входа, инициализация Loguru и запуск цикла Qt
│   ├── api/                       # Сетевой уровень (Network Layer)
│   │   ├── __init__.py
│   │   └── client.py              # HTTP-клиент (httpx), обработчик сетевых статусов и валидатор структур
│   ├── core/                      # Ядро системы и конфигурации
│   │   ├── __init__.py
│   │   └── config.py              # Инициализация логгера, константы пресетов колонок (Базовый/Проф)
│   ├── models/                    # Аналитический и модельный уровень (Model Layer)
│   │   ├── __init__.py
│   │   └── table_model.py         # [В разработке] Кастомная Qt-модель данных для сопряжения Pandas и QTableView
│   └── views/                     # Графический интерфейс (View Layer)
│       ├── __init__.py
│       ├── filter_panel.py        # [В разработке] Панель управления фильтрами и валидация ввода
│       └── main_window.py         # Главное окно графического интерфейса (PySide6)
├── tests/                         # Модульное и интеграционное тестирование (Тестовое окружение)
│   ├── __init__.py
│   └── test_api.py                # Юнит-тесты сетевого клиента, мокание аномалий сети и проверка фикстур
├── .gitignore                     # Список исключений для системы контроля версий Git
├── poetry.lock                    # Фиксация точных версий зависимостей проекта
├── pyproject.toml                 # Главный манифест проекта (настройки Poetry, Ruff линтера и Pytest)
└── README.md                      # Документация проекта, руководство по запуску и отчетность
